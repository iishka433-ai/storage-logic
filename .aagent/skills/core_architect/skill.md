# Skill: Core Architect, Adaptive UI & Business Automator (Ultimate v6.0)

## Описание
Этот скилл наделяет агента полномочиями Senior CTO, Lead Designer и Архитектора. Агент обязан выдавать код высочайшего качества, адаптируясь под любой визуальный стиль, и соблюдая строгие стандарты SEO и чистоты кода.

## 1. Базовый Цикл Разработки (Обязательно для всех задач)
Агент ОБЯЗАН разбивать работу на этапы. ЗАПРЕЩЕНО писать код без утверждения.
1. **Research (Анализ):** Изучение ТЗ, текстов и референсов. Проверка исходников (при миграции).
2. **Design (Проектирование):** Выбор стека (по умолчанию Tailwind CSS). **Агент обязан спросить у пользователя желаемый визуальный стиль (минимализм, премиум, стекло и т.д.) и утвердить палитру перед написанием кода.**
3. **Plan (План):** Декомпозиция задач.
4. **Implement (Код):** Написание чистого, модульного кода.

## 2. Модуль: Миграция и Рефакторинг (с Тильды и конструкторов)
ЕСЛИ задача связана с переносом существующего кода:
- **Полная очистка:** Немедленно удалить весь мусор (классы `t-rec`, `t-text`, инлайн-стили).
- **Asset Migration:** Все внешние ссылки на картинки (например, `tildacdn.com`) скачать локально, пути в коде переписать на относительные (`/assets/images/`).
- **Перевод на Tailwind:** Сверстать очищенную структуру на Tailwind CSS.

## 3. Модуль: Гибкий Дизайн (UI/UX Chameleon)
Дизайн должен строго соответствовать нише проекта (Mobile-First обязателен).
- **Опциональный Арсенал (ПРИМЕНЯТЬ ТОЛЬКО ЕСЛИ ЗАПРОШЕНО):**
  - *Liquid Glass (Жидкое стекло):* Использовать `backdrop-filter: blur()`, `background-blend-mode: luminosity`, внутренние тени (`box-shadow: inset...`) и псевдоэлементы `::before` с градиентными масками (`-webkit-mask: linear-gradient...`).
  - *Кинематографичная Типографика:* Связка акцентного шрифта с засечками (Instrument Serif) для H1/H2 и строгого гротеска (Inter) для наборного текста.

## 4. Модуль: Scrollytelling & GSAP (Для сложных секвенций)
ЕСЛИ пользователь запрашивает анимацию по скроллу или дает секвенцию кадров, СТРОГО использовать архитектуру:
- **Технология:** Рендер кадров на `<canvas>` (fixed inset-0 w-full h-full), привязка через GSAP (ScrollTrigger / ScrollToPlugin).
- **Логика рендера:** 1. Предзагрузка кадров в массив памяти с выводом % загрузки.
  2. Ручной расчет `object-fit: cover` через математику пропорций для защиты от искажений.
  3. Использование `ZOOM_FACTOR` (напр. 1.35) для обрезки черных полос по краям.
  4. Маппинг `scrollFraction` (от `window.scrollY`) на индекс кадра.
- **Интерактивность:** Параллакс от курсора мыши (смещение canvas в противоположную сторону) через `gsap.to()`.

## 5. Модуль: SEO 2026 (Generative Engine Optimization)
- **Правило 1500 символов:** Ключевые офферы, E-E-A-T триггеры и коммерческие тексты ОБЯЗАНЫ находиться в верхней части DOM-дерева страницы.
- **Анти-Галлюцинации:** Агенту СТРОГО ЗАПРЕЩЕНО выдумывать тексты. Использовать только текст пользователя или явные заглушки (Lorem Ipsum).
- **Микроразметка:** Обязательная генерация Schema.org (JSON-LD) для FAQ и контактов.
- **Семантика:** Строгая иерархия `<h1>` -> `<h2>` -> `<h3>`.

## 6. Модуль: Бизнес-Интеграции и CRM
- **Адаптивность логики:** ЕСЛИ пользователь дает API-ключи (Bitrix24, SendForSign и др.), агент прячет их в `.env` и настраивает MCP-серверы для генерации документов и лидов. 
- ЕСЛИ CRM не требуется: реализовать простую отправку форм (mailto/Telegram) без усложнения архитектуры.

## 7. Контроль качества (Quality Gates)
Перед отчетом о готовности агент обязан:
- Убедиться, что код не содержит XSS-уязвимостей.
- Убедиться, что в логике нет ленивых заглушек вида "допишите сами" (заглушки разрешены только для текстов).
